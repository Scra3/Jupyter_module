• /* READ ME INSTALLATION DE SAGE */



1-Télécharger le bon fichier sage. Seulement la version 7.3 et faire très attention a votre distribution de Unix. Sinon ça va tout gacher.

1Go d'espace
http://www.sagemath.org/download-source.html


2-Vérifier que vous possédez les applications nécessaires pour pouvoir Make le fichier:

Indispensable : 
sudo apt-get install binutils gcc make m4 perl tar git
sudo apt-get install binutils gcc g++ gfortran make m4 perl tar git
sudo apt-get install libssl libssl-dev


facultatif mais rajoute des animations
sudo apt-get install texlive   


3-Déplacer le fichier télécharger dans un répertoire, personnellement, je l'ai mis dans /home.
ATTENTION : il ne doit y avoir aucun accent dans les dossiers dans lequel il se trouve. 
En faite, le chemin ca etre stocké dans une variable, et python ne reconnait pas de base les caracteres speciaux, a moins de rajouter le format utf-8 dans chaqu'un des fichiers pythons qui sont trop nombreux.


4-Ensuite se placer dans le répertoire et décompresser avec : 
tar xvf nomdufichier

Cette opération prend un moment, soyez patient.



5-Déplacez vous dans le répertoire SageMath, et lancez la commande Make.

Attention cette opération prend énormément de temps. Sur mon PC portable (i3,4go de ram) ça m'a pris exactement 2h27 minutes. Prévoyez un film xD


7- Se placer dans sageMath et faire ./sage, et normalement le truc se lance

8- ecrire quit pour quitter, et lancer ./sage -n jupyter pour pouvoir lancer jupyter


Note importante : Vous ne pouvez faire l'installation qu'une seule fois, sinon il faut recommencer à partir de l'étape 3. Il  y a un script qui installe les binary qui ne se fait qu'une seule fois.. Si jamais vous rencontrez des erreures, il faut repérer les erreur d'installations et les régler dans le terminal.


Trucs fréquents :
- sage requiere ssl instead of HTMLS ou un truc du genre  = probleme lors de l'install de open-ssl
-permission denieds = prob de permission = repérer le fichier et faire un chmod a+r du fichier.
ou chmod - R a+r nomrep pour appliquer ça de façon recurisve a tous le contenu du rep
SI jamais le dossier commence par .NOMREP c'est que c'est un fichier caché, et ils ne sont pas affectés par chmod. Dans ces cas la utiliser chown a+r ou chown -R


Pour d'autres erreurs, on verra ça ensemble lundi
